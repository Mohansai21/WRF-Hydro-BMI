### ğŸ”‘ The Five Sacred Separator Characters

Standard Names use exactly 5 special characters, each with a precise meaning:

| Separator | Meaning | Example |
|-----------|---------|---------|

| `~` (tilde) | Joins adjective to noun (noun first) | `water~incoming` |
| `_of_` | Applies a math operation | `time_derivative_of_mass` |

### ğŸ“ The Case Rule

```

âŒ  CHANNEL_WATER__VOLUME_FLOW_RATE
```

The only allowed characters are: `a-z`, `0-9`, `_`, `-`, `~`

### ğŸ“ The Regex (for the technically curious)

The official regular expression that validates any CSDMS Standard Name:

This enforces: starts with lowercase, has exactly one `__`, and ends with a letter or number.


## 8ï¸âƒ£ How Standard Names Compare to CF Standard Names

### ğŸ“Š CSDMS vs CF Convention

Many people know the CF (Climate and Forecast) Standard Names used in NetCDF files. CSDMS Standard Names were designed to improve on CF:

| Feature | CF Standard Names | CSDMS Standard Names |
|---------|------------------|---------------------|
| Structure | Flat string, no explicit object/quantity split | Always object__quantity |
| Domain | Mostly ocean & atmosphere | Cross-domain (any science) |
| Rules | Incomplete guidelines | Strict, parseable rules |
| Constants | No support | Supported |
| Dimensionless numbers | No support | Supported (e.g. Reynolds number) |
| Math operations | Uses "tendency_of" (domain-specific) | Uses "time_derivative_of_" (generic) |
| Assumptions | Embedded in name | Separate metadata (MCM file) |
| Total names | ~2,500 | Growing registry |
| Parseable | Difficult | Yes (regex-validated) |

**Example comparison:**

```
CF:    air_temperature
CSDMS: atmosphere_bottom_air__temperature

CF:    surface_runoff_flux
CSDMS: land_surface_water__runoff_volume_flux

CF:    sea_surface_height_above_geoid
CSDMS: sea_water_surface__elevation
```

CSDMS names are more verbose but much more precise and parseable.

### âš¡ Quick Tips

1. **Search first, create second.** Always check the existing registry before making up a new name
2. **Max 4 adjectives.** Usually no more than 4 qualifier words are needed to remove ambiguity
3. **General to specific.** Build from left to right: `land_surface_air__temperature` (land â†’ surface â†’ air)
4. **Keep objects out of quantities.** Don't say `water_temperature_of_ocean`. Say `sea_water__temperature`
5. **Use existing chemistry names.** For chemical species, follow IUPAC conventions
6. **Units are separate.** Units are NOT part of the name â€” they go in the BMI's `get_var_units()` function
